# Modules

Modules are the core features of Aoba Client. Each module can be individually enabled or disabled, providing specific 
functionality such as flying, ore detection, combat assistance, and more.

## Toggling Modules

Modules can be toggled in four ways:

- **ClickGUI** - Open the [ClickGUI](/wiki/basics/clickgui) and click a module to toggle it on or off. Click the gear icon next to a module to access its settings.
- **Module Selector HUD** - Use the on-screen module selector for quick access.
- **Chat Commands** - Toggle modules directly via chat commands.
- **Keybinds** - Assign a keybind to any module for instant toggling without opening the GUI.

![ClickGUI Modules](/screenshot_clickgui_modules.png)

## Module Settings

Each module has its own configuration that can be accessed through the [ClickGUI](/wiki/basics/clickgui). Next to every module name is a **gear icon** that opens 
a dedicated settings window for that module. These settings allow you to fine-tune the module's behavior, such as adjusting ranges, speeds, targets, and other module-specific options. 
All settings persist across game sessions, so you only need to configure them once.

![Module Settings](/screenshot_config.png)

```
⚠ Disclaimer: Most servers run anti-cheat plugins. Modules must be properly tuned through their settings to avoid detection. 
Be aware of which modules you are using and adjust their values accordingly to minimize the risk of bans.
```

## Anticheat Window

The ClickGUI includes an **Anticheat Window**, which contains a list of known server-side anticheats. Each anticheat entry displays 
all modules that are detectable by that specific anticheat plugin. When you select an anticheat from the list, Aoba will automatically 
disable any modules that are known to be flagged by it, helping to protect you from accidental bans.

Simply select the AntiCheat that the server is running and Aoba will handle the rest by 
restricting modules that would otherwise get you detected. You can switch between anticheats at any time, and the module restrictions will update accordingly.
